Security is critical for any fintech integration. Follow these guidelines to keep your users and your app safe.

## API Key storage

::caution
Never hardcode your API secret key (`sk_live_...`) directly in your Dart source code. It can be easily extracted from your compiled `.apk` or `.ipa` app through reverse-engineering, giving attackers full access to your FedaPay account.
::

### 1. Recommended: Backend Proxy (Frontend-Only Integration)

For maximum security in production, your Flutter app should **never** know the FedaPay secret key. 

Instead of configuring the SDK with your API key, your mobile app should only use the `PayWidget` to display a transaction generated by your own backend:

1. Your Flutter app calls **your own backend/API**.
2. Your backend calls FedaPay Server-to-Server using the secret key (`sk_live_...`) to create a transaction.
3. Your backend returns the **Transaction Token** to the mobile app.
4. You pass the `transactionToken` directly to the `PayWidget`:

```dart [main.dart]
// No need to initialize FedaFlutter.applyConfig()!

PayWidget(
  transactionToken: 'token_from_your_backend', // e.g., "tr_..."
  onPaymentSuccess: () { print('Success!'); },
  onPaymentFailed: () { print('Failed!'); },
)
```

This is the **only architecture recommended for production**. It prevents any risk of key leakage from your mobile app.

### 2. Alternative: Environment Variables (Testing / Prototyping)

If you are prototyping or building an internal app and need the app to create transactions directly, pass your key at build time without committing it to your repository:

```bash [Terminal]
flutter run --dart-define=FEDAPAY_API_KEY=your_key_here
```

Then read it in Dart and initialize the SDK:

```dart [main.dart]
const apiKey = String.fromEnvironment('FEDAPAY_API_KEY');

// ⚠️ WARNING: The key is still embedded in the binary!
FedaFlutter.applyConfig(
  apiKey: apiKey,
  environment: ApiEnvironment.sandbox,
);

// Now you can use the repositories, or the PayWidget with an instance
final instance = FedaFlutter.instance;
```

## Sandbox vs Live

| | Sandbox | Live |
|---|---|---|
| Key prefix | `sk_sandbox_...` | `sk_live_...` |
| Real money | ❌ | ✅ |
| Use for | Development & testing | Production |

```dart
// Development
FedaFlutter.applyConfig(
  apiKey: 'sk_sandbox_...',
  environment: ApiEnvironment.sandbox,
);

// Production
FedaFlutter.applyConfig(
  apiKey: 'sk_live_...',
  environment: ApiEnvironment.live,
);
```

::tip
Use [Flutter flavors](https://docs.flutter.dev/deployment/flavors) or `--dart-define-from-file` to switch environments automatically between debug and release builds.
::

## Going live checklist

Before switching to `ApiEnvironment.live`:

- [ ] Tested all payment flows in sandbox
- [ ] API key stored securely (not hardcoded)
- [ ] Error handling implemented (`onPaymentFailed`, `onPaymentCanceled`)
- [ ] HTTPS enforced on your backend (if using proxy)
- [ ] FedaPay account verified and live mode enabled on your dashboard
